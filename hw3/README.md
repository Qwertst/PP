Реализовать потокобезопасный IntSet на основе связного списка.

Последовательно реализовать 4 стратегии синхронизации:

    Грубая: Один мьютекс на весь список.

    Тонкая: Мьютекс на каждый узел (захват при проходе + ручная блокировка next перед удалением).

    Оптимистичная: Поиск без блокировок, затем захват и валидация.

    Ленивая: Пометка узла как удаленного (флаг) физическое удаление позже.

Исследование:

    Сравнить производительность (операций/сек) при 20% записи / 80% чтения.
    Отчет: Вывод, при каких условиях «ленивая» синхронизация выигрывает.
