# Задача 2

Каждая итерация выполняет три отдельные операции:
1. `temp = count`
2. `temp + 1`
3. `count = temp + 1`

### Максимум: **10,000** 

Все операции выполняются последовательно без наложения 

### Минимум: **100**

Каждая из 100 итераций (по одной от каждого потока) должна увеличить `count` хотя бы на 1, даже при максимальном наложении. Невозможно, чтобы итерация не увеличила значение вообще, так как:
- Хотя бы последний поток, записывающий значение, увеличит `count` на 1
- Минимум 100 итераций (по одной на поток) должны пройти

Например если все потоки читают одно значение перед записью в каждой итерации
